## Executive summary

The pipeline is **technically healthy** (65/65 tests, benchmark green, validation loop demonstrably working — 11 real classifier improvements shipped in 4 days). The **documentation has drifted hard**: README claims a sidebar source toggle that no longer exists, lists Fig 4's forest plot that was deleted, omits the 6 antigens added today and 30+ commits' worth of features. The **auto-generated Methods text** that users copy into their papers under-counts heme antigens (`16` shown, `22` actual) and solid antigens (`25` vs `28`), referencing a curation workflow (`validate.py`) that has been superseded but not removed. The **single snapshot** at `2026-04-24/` is the entire reproducibility surface — there's no rollback path if a classifier change cascades badly.

**Top 3 issues**: (1) Methods text drift creates *citable* misinformation; (2) `requirements.txt` is missing `google-genai` and `groq` so a fresh checkout can't run the validation harness; (3) the `validate.py` / `scripts/validate_independent_llm.py` duality is unmarked, leaving newcomers to guess which is canonical.

**Top 3 strengths**: (1) The validation feedback loop is a genuine engine — disagreement bucket has shrunk meaningfully across iterations; (2) Caching architecture (session_state + `@st.cache_data` + `@st.fragment`) is correctly layered, no obvious remaining hot paths; (3) Style-guide compliance is clean (no purple/violet/indigo anywhere except documenting comments).

Closes the silent-coverage-gap implication of risk #6. ~3 days including triage.

### Phase 3 — Steady-state (cadence, monitoring, automation)

These are the rituals, not the projects.

- **Weekly**: independent-LLM run (50 trials min, both providers when quota allows), triage consensus disagreements within 24h, ship fixes with regression tests.
- **Monthly**: snapshot save → snapshot-diff against previous month → audit "unexplained" reclassifications. Quarterly approvals review per the existing GitHub issue template.
- **Per-PR (CI)**: `pytest tests/ -q` must pass; `pytest tests/test_benchmark.py` per-axis F1 floors enforced.
- **Per-snapshot**: provenance header includes classifier git SHA; pinned ontology copy alongside CSV.
- **Per-release** (i.e., what counts as a Zenodo version bump): CHANGELOG updated, version bumped in CITATION.cff, tag pushed, Zenodo auto-archives.
- **Quarterly**: README + Methods text proofread against current code (or, ideally, generate them from code so this isn't needed).
